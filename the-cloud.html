Analisis de codigo

 

Autenticación de usuarios:

  1. La Conexión (src/supabaseClient.ts)

Este es el puente con la nube (en mi caso Supabase). Aquí se inicializa la librería de Supabase con las credenciales.

  1. El Cerebro (src/context/AuthContext.tsx)

Este archivo es el más importante. Crea un "Contexto" (un estado global) que envuelve a toda la aplicación para que cualquier componente sepa si hay un usuario logueado.

Estado Global: Guarda el user y session.

Listener: En el useEffect, se suscribe a Supabase (onAuthStateChange) para detectar automáticamente si el usuario inicia sesión, cierra sesión o refresca la página.

  1. La Interfaz (src/pages/Login.tsx)

Es el formulario visual. No tiene lógica compleja, solo recoge los datos y llama a las funciones del AuthContext.

Usa un estado isLogin para alternar entre "Iniciar Sesión" y "Registrarse".

Cuando se envia el formulario:

  1. El Guardián (src/App.tsx)

Aquí es donde se protege la navegación. Usamos dos componentes especiales:

PrivateRoute: Si NO hay usuario, te manda al Login. Si hay usuario, te deja pasar al Dashboard.

PublicRoute: Si YA hay usuario (ej. intentas entrar a /login estando logueado), te manda directo al Dashboard.

 

Preview archivos y editores:

 

1. Previsualización de Archivos
La lógica principal de previsualización se encuentra en el archivo src/components/FilePreviewModal.tsx


Documentos de Word (.docx):
Librería: mammoth
Uso: Se usa para convertir el archivo .docx a HTML para poder mostrarlo.

Hojas de cálculo (.xlsx):
Librería: xlsx (SheetJS)
Uso: Lee el archivo Excel y convierte la primera hoja a HTML.

Archivos Markdown (.md):
Librería: react-markdown
Uso: Renderiza el contenido Markdown como HTML visualizable.

PDF e Imágenes:
Sin librería externa: Usa las capacidades nativas del navegador (<iframe> para PDF e <img> para imágenes) cargando la URL firmada de Supabase.


Código (.js, .ts, .java, etc.):
Sin librería externa en el modal: En la ventana modal de previsualización, el código se muestra simplemente dentro de una etiqueta <pre><code> (líneas 159-162), sin usar un editor complejo.

2. Modificación / Edición de Archivos
La lógica de edición se encuentra en src/pages/FileEditor.tsx, que es la página a la que navegas al pulsar "Editar".

Archivos de Código (.ts, .js, .json, .java, etc.):
Librería: @monaco-editor/react (El mismo editor que usa VS Code).
Uso: Provee un editor de código completo con coloreado de sintaxis.

Archivos Markdown (.md) - Previsualización en el Editor:
Librería: react-markdown
Uso: Permite ver cómo quedará el documento mientras lo editas (modo "Preview").

Archivos Markdown y Texto Plano (.txt) - Edición:
Nativo: Para la escritura en sí de estos formatos, se utiliza un elemento <textarea> estándar de HTML manipulado con React (líneas 201 y 213), apoyado por funciones auxiliares personalizadas para la barra de herramientas de formato (negrita, cursiva, etc.).
